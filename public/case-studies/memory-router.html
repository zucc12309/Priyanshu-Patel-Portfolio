Memory Router

I built a local-first intelligent model routing system with structured memory, semantic retrieval, context optimization, adaptive provider routing, and MCP integration.

Problem

LLM workflows resend too much context, lose durable memory, and force users to choose between privacy and capability.

Solution

I built a local-first memory and routing layer that retrieves relevant knowledge, assembles compact prompts, and sends work to the right provider.

Architecture

CLI / MCP Server → Context Builder → Memory Palace + FTS5 → Vector Store → Adaptive Router → LLM Providers.

Impact

I used README examples to target 80-90% input token savings while preserving local ownership of memory.
