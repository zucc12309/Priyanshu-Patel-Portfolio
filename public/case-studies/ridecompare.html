RideCompare

I built a production-grade mobile MVP for comparing ride fares across providers and choosing the best option.

Problem

Users manually check multiple ride apps without knowing which provider is cheapest, fastest, or most reliable.

Solution

I shaped a Flutter app plus Node backend that estimates fares, ranks options, tracks actual paid fares, and deep-links to providers.

Architecture

Flutter app → Auth/session store → Node API → Fare engine → Maps proxy → PostgreSQL → Analytics feedback loop.

Roadmap

I would expand this with live integrations, personalized savings, city expansion, and native booking handoff.
